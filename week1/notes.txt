# 🐍 Python Roadmap — Day 1 Notes

## 📁 Why split code into multiple files?

Each file should have **one responsibility**.

```
main.py              → Starts the program
student_manager.py   → Student-related logic
models.py            → Defines student data
storage.py           → Reads/Writes JSON files
students.json        → Stores the data
```

This makes projects easier to read, debug, and maintain.

---

# 📦 Imports

Use imports to reuse code from another file.

```python
from models import create_student
```

Instead of rewriting functions, simply import them.

---

# 📄 JSON

JSON is a lightweight format for storing data.

Example:

```json
[
    {
        "name": "Haseeb",
        "age": 21,
        "department": "Software Engineering"
    }
]
```

Python converts JSON into normal lists and dictionaries.

---

# 📂 pathlib

```python
from pathlib import Path
```

Instead of hardcoding file paths, use `Path()`.

Example:

```python
DATA_FILE = Path("data/students.json")
```

Benefits:

* Works on Windows, Linux, and macOS
* Cleaner and easier to read

# 🔄 json Module

## json.load(file)

Reads JSON **from a file** and converts it into Python objects.

```
JSON File
    ↓
json.load()
    ↓
Python List / Dictionary
```

---

## json.dump(data, file)

Converts Python objects into JSON and writes them to a file.

```
Python List / Dictionary
    ↓
json.dump()
    ↓
JSON File
```

# ⭐ Important Line Explained

```python
DATA_FILE.parent.mkdir(exist_ok=True)
```

Breakdown:

* `DATA_FILE` → `data/students.json`
* `.parent` → `data/` folder
* `.mkdir()` → Create the folder
* `exist_ok=True` → Don't raise an error if it already exists

Meaning:

> **Create the `data` folder if it doesn't already exist.**

# 🐍 Python Roadmap — Day 4 Notes

## 🛡️ Input Validation

Never trust user input! Users can enter empty strings, letters instead of numbers, or negative values.

We learned to use `while True:` loops to keep asking for input until valid data is provided.

```python
while True:
    name = input("Enter name: ").strip()
    if name == "":
        print("Name cannot be empty")
        continue
    else:
        break
```

## 🪲 Try and Except

Used to prevent the program from crashing when converting data types (like `int("abc")`).

```python
while True:
    try:
        age = int(input("Enter age: "))
        if age <= 0 or age > 120:
            print("Invalid age")
        else:
            break 
    except ValueError:
        print("Enter integer only")
```

## 📝 Helper Functions

Instead of repeating the same `input()` code in multiple places, we extracted it into a single helper function `get_student_details()`. This follows the DRY (Don't Repeat Yourself) principle.

## 📖 Docstrings

A short explanation of what a function does. It goes right under the `def` line and uses triple quotes `"""`.

```python
def search_student(name):
    """Searches for a student by their name and returns the student dictionary."""
```

## 🏷️ Type Hints

Helps editors and other developers understand what type of data is expected.

* Parameters: `name: str`
* Return value: `-> dict`

```python
def search_student(name: str) -> dict:
```

## 💡 Professional Habits Learned Today

* **Fail gracefully**: Don't let bad input crash your app.
* **DRY Code**: Extract repetitive logic into helper functions.
* **Document code**: Use docstrings and type hints to make code readable for others.
